Building This Website

A guide to building this very site.

Absolutely — here’s the revised full guide for setting up a Hugo + PaperMod site using only GitHub (no local Hugo installation), fully automated with GitHub Actions and a personal access token (PAT) for cross-repo deployment.

🧱 Prerequisites#

  • GitHub account

  • Two GitHub repositories:

    • hugo-alexbakertech → Hugo source repo (can be private)

    • alexbakertech.github.io → GitHub Pages deployment repo (must be public)

  • Optional: a custom domain like alexbakertech.com

🧰 Step 1: Set Up Your GitHub Repos#

A. Create Repositories#

  1. Create hugo-alexbakertech
    This holds your content and build configuration.

  2. Create alexbakertech.github.io
    This will host your final website via GitHub Pages. Leave it empty — GitHub Actions will push content into it.

B. Add Hugo + PaperMod Files to hugo-alexbakertech#

Use the GitHub file editor or GitHub.dev (. shortcut) to create the following structure:

hugo-alexbakertech/
├── config.toml
├── content/
│   └── posts/
│       └── first-post.md
├── themes/
│   └── PaperMod/    ← paste PaperMod files here
└── .github/
    └── workflows/
        └── deploy.yml

config.toml (site config)#

baseURL = "https://alexbakertech.com/"
languageCode = "en-us"
title = "alexbakertech"
theme = "PaperMod"

[params]
  author = "Alex Baker"
  ShowReadingTime = true
  defaultTheme = "auto"

content/posts/first-post.md#

---
title: "My First Post"
date: 2025-06-27
draft: false
---

This is my first post using Hugo and GitHub-only deployment.

themes/PaperMod/#

Copy the contents of the PaperMod GitHub repo into this folder using the GitHub file interface or GitHub.dev.

🔐 Step 2: Create a Personal Access Token (PAT)#

  1. Go to: https://github.com/settings/tokens

  2. Click “Generate new token (classic)”

  3. Name: HugoDeployToken

  4. Set expiration and check:

    • ✅ repo scope

  5. Generate and copy the token — you’ll use it once to configure secrets

🔐 Step 3: Add the Token as a GitHub Secret#

  1. In the hugo-alexbakertech repo, go to:
    Settings → Secrets and variables → Actions → New repository secret

  2. Add a secret:

    • Name: PERSONAL_TOKEN

    • Value: (paste your token)

⚙️ Step 4: Add GitHub Actions Workflow#

Create .github/workflows/deploy.yml in the hugo-alexbakertech repo:

name: Deploy Hugo site to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout source
        uses: actions/checkout@v3

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'

      - name: Build site
        run: hugo --minify

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          personal_token: ${{ secrets.PERSONAL_TOKEN }}
          publish_dir: ./public
          external_repository: alexbakertech/alexbakertech.github.io
          publish_branch: main
          cname: alexbakertech.com

🔁 Every push to main will trigger this workflow and deploy the site to alexbakertech.github.io.

🌍 Step 5: Configure GitHub Pages for Your Domain#

In the alexbakertech.github.io repo:

  1. Go to Settings → Pages

  2. Set:

    • Source: main branch, / (root)

    • Custom domain: alexbakertech.com (optional)

    • ✅ Enforce HTTPS

🌐 Step 6: Set DNS (for alexbakertech.com)#

In your domain provider’s DNS panel, add the following A records:

TypeNameValue
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153

Optional CNAME:

www → alexbakertech.github.io

🧪 Step 7: Test It#

  1. Make a content change in hugo-alexbakertech (e.g., edit first-post.md)

  2. Commit and push the change to the main branch

  3. Go to Actions tab to confirm the build runs

  4. Visit:

    • https://alexbakertech.com (if custom domain is set up)

    • Or https://alexbakertech.github.io

✅ You’re Done!#

You now have:

  • A GitHub-only Hugo site using PaperMod

  • No local install needed

  • Private source repo (optional)

  • Auto-deploys to GitHub Pages using a secure personal access token
