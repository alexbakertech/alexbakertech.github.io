How the alexbakertech.com site is built and deployed using GitHub Actions and GitHub Pages

🧭 Overview#

This site is a static Hugo build served through GitHub Pages using a GitHub Actions workflow for automated deployment. The project uses the PaperMod theme with custom branding, content structure, and image handling logic.

🧱 Repository Structure#

The site is stored in a private repository named hugo-alexbakertech, and follows this structure:


/
├── content/
├── static/
├── layouts/
├── assets/css/extended/
├── config.toml
└── .github/workflows/deploy.yml
  • /content: Markdown content (posts, about, contact, projects)
  • /static: Image assets (favicon, banner, logos)
  • /layouts: Template overrides for Hugo and PaperMod
  • /assets/css/extended: Custom styles (theme overrides, layout tweaks)
  • config.toml: Site configuration and global settings
  • deploy.yml: CI pipeline to build and publish the site

⚙️ GitHub Actions Deployment#

The CI/CD pipeline uses GitHub Actions to build the Hugo site and deploy it to GitHub Pages. The deploy.yml file is stored in:


.github/workflows/deploy.yml

🔐 Key Setup Notes#

  • You must place the file in .github/workflows/, not .git/workflows/
  • Ensure the repo’s GitHub Pages setting points to gh-pages branch with Deploy from a branch
  • The Action uses the github_token to authenticate deployment

🛠 Sample Workflow File#

name: Deploy Hugo site to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

🌐 Custom Domain Configuration#

  • The domain alexbakertech.com is connected via GitHub Pages settings
  • Hugo’s baseURL in config.toml is set to the full domain:
baseURL = "https://alexbakertech.com/"

📌 Deployment Checklist#

  • Initialize Hugo site in repo
  • Add PaperMod theme as a Git submodule
  • Place site assets in /static
  • Configure GitHub Pages (gh-pages branch)
  • Add deploy.yml to .github/workflows
  • Push to main branch to trigger deployment

✅ Outcome#

With every push to main, GitHub Actions builds and deploys the latest version of the site. No local Hugo installation is required — builds happen entirely in the cloud, ensuring consistency and reducing setup overhead.
