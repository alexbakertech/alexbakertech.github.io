3ï¸âƒ£ Add GitHub Actions Deployment Workflow#

In .github/workflows/deploy.yml:

name: Deploy Hugo site to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'

      - name: Build site
        run: hugo --minify

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
          external_repository: alexbakertech/alexbakertech.github.io
          publish_branch: main
          cname: alexbakertech.com

6ï¸âƒ£ Push to Deploy#

Any push to main in hugo-alexbakertech triggers the workflow:

  • Builds the Hugo site
  • Pushes to alexbakertech.github.io
  • Publishes to https://alexbakertech.com

You can monitor deployments in the Actions tab.

ğŸ§  Optional: One-Repo Alternative#

You can deploy directly from a gh-pages branch in a single repo:

  • Remove external_repository from the workflow
  • Set publish_branch: gh-pages
  • Configure GitHub Pages to serve from gh-pages

This is simpler for hobby use but less modular.
