🧭 Overview#

This guide walks through how this site — alexbakertech.com — is built and deployed using only GitHub tools. It uses:

  • A private Hugo source repo
  • A public GitHub Pages repo
  • GitHub Actions for automated deployment
  • No local Hugo installation

📦 Prerequisites#

  • A GitHub account
  • Two repositories:
    • hugo-alexbakertech → Source repo (private)
    • alexbakertech.github.io → Deployment repo (public)
  • A custom domain (e.g., alexbakertech.com) configured with A records

1️⃣ Create the Repositories#

A. Source Repo: hugo-alexbakertech#

Holds all Hugo content, theme, config, and build logic. This repo remains private.

B. Deployment Repo: alexbakertech.github.io#

A public repo used for GitHub Pages hosting. Content is deployed here automatically — no manual updates required.

2️⃣ Add Hugo & PaperMod to the Source Repo#

Use GitHub.dev (.) or the file editor to scaffold:

3️⃣ Add GitHub Actions Deployment Workflow#

In .github/workflows/deploy.yml:

name: Deploy Hugo site to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'

      - name: Build site
        run: hugo --minify

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
          external_repository: alexbakertech/alexbakertech.github.io
          publish_branch: main
          cname: alexbakertech.com

✅ Uses secure GITHUB_TOKEN ❌ No personal access token required

4️⃣ Configure GitHub Pages (Deployment Repo)#

In the alexbakertech.github.io repo:

  1. Go to Settings → Pages

  2. Set:

    • Source: main branch → / (root)
    • Domain: alexbakertech.com
    • ✅ Enforce HTTPS

Optional CNAME:

www → alexbakertech.github.io

6️⃣ Push to Deploy#

Any push to main in hugo-alexbakertech triggers the workflow:

  • Builds the Hugo site
  • Pushes to alexbakertech.github.io
  • Publishes to https://alexbakertech.com

You can monitor deployments in the Actions tab.

🧠 Optional: One-Repo Alternative#

You can deploy directly from a gh-pages branch in a single repo:

  • Remove external_repository from the workflow
  • Set publish_branch: gh-pages
  • Configure GitHub Pages to serve from gh-pages

This is simpler for hobby use but less modular.
