🧭 Overview#

This document explains the internal structure of alexbakertech.com, including how content is organized, rendered, and extended using Hugo and the PaperMod theme. It covers:

  • Page types and folder layout
  • Use of leaf bundles for projects
  • Frontmatter conventions
  • TOC behavior
  • Menu and navigation logic

🗂️ Content Folder Layout#

The content/ folder is divided into:


content/
├── \_index.md               → Homepage text + metadata
├── about.md                → Static about page
├── contact.md              → Static contact page
├── posts/                  → Blog-style entries
│   └── my-first-post.md
└── projects/               → Project documentation
├── \_index.md
└── secure-home-lab/
├── \_index.md       → Project overview
├── architecture.md
└── deployment.md

Project folders are leaf bundles, allowing each project to have multiple Markdown subpages and image assets.

🧱 Page Types#

📌 Frontmatter Conventions#

Each page uses Hugo’s TOML/YAML frontmatter to control behavior:

---
title: "Secure Home Lab"
description: "Planning a hardened, private lab environment"
showToc: true
draft: false
---

Common fields:#

KeyPurpose
titlePage title
descriptionUsed for metadata and SEO
showTocEnables floating Table of Contents (default is off)
draftHides content from build output

🧭 TOC Behavior#

Global Setting (in config.toml):#

[params]
  ShowToc = false  # Off by default
  TocOpen = true   # Expanded when shown

Per-Page Override:#

To enable TOC for a single page:

showToc: true

⚠️ TOC does not render on _index.md files unless custom layout logic is added. This is by design.

🧩 Project Pages as Leaf Bundles#

Each project has its own folder (e.g. projects/secure-home-lab/) containing:

  • _index.md: A short intro or summary
  • *.md: Topical subpages with TOC support
  • /images/ (optional): Images specific to that project

This pattern is scalable, documentation-friendly, and keeps URLs clean:

/projects/secure-home-lab/overview/
/projects/secure-home-lab/networking/
/projects/secure-home-lab/deployment/

🧭 Navigation Logic#

Navigation is centralized in config.toml:

[[menu.main]]
  name = "About"
  url = "/about/"
  weight = 1
  • Menu order is controlled by weight
  • Pages do not define their own menu items in frontmatter
  • Projects are not linked automatically — they’re accessed from the Projects index

🧹 Notes and Design Choices#

  • Avoided taxonomy (tags/categories) to reduce clutter
  • TOC is opt-in to avoid visual overload on short pages
  • Project sections are designed to support longform, modular content
  • All extended layouts live in /layouts/ for maintainability

✅ Summary#

This structure balances scalability, readability, and ease of contribution. It can support personal posts, professional documentation, tutorials, or full technical case studies.

Future projects can simply duplicate an existing leaf bundle as a template — just change the frontmatter and drop in the new Markdown content.
