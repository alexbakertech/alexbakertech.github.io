🧭 Overview#

This blueprint outlines how to replicate the Hugo structure used for alexbakertech.com, including:

  • Project and content layout
  • TOC-enabled Markdown pages
  • GitHub Actions deployment
  • PaperMod theme with clean overrides

It is ideal for technical portfolios, documentation hubs, or longform reference material.

🗂️ Folder Layout#

Here’s the minimal base structure you can reuse:


/
├── content/
│   ├── about.md
│   ├── contact.md
│   ├── posts/
│   │   └── first-post.md
│   └── projects/
│       ├── \_index.md
│       └── my-project/
│           ├── \_index.md
│           ├── overview\.md
│           └── deployment.md
├── static/
│   └── (banner, favicons, etc.)
├── assets/css/extended/
│   ├── theme-vars.css
│   └── about.css
├── layouts/
│   ├── index.html
│   └── partials/
│       ├── extend\_head.html
│       └── extend\_footer.html
├── .github/workflows/deploy.yml
└── config.toml

🧱 Page Structure Patterns#

A. Projects as Leaf Bundles#

Each project is a folder with multiple .md subpages and optional image assets:


content/projects/my-project/
├── \_index.md      → project summary
├── overview\.md    → technical write-up
└── architecture.md → diagrams or analysis

Use showToc: true in frontmatter to enable navigation sidebar on subpages.

🧩 Key Configuration Snippets#

config.toml – Content & Theming#

baseURL = "https://example.com/"
languageCode = "en-us"
title = "My Hugo Site"
theme = "PaperMod"

[params]
  author = "Your Name"
  defaultTheme = "dark"
  ShowToc = false
  TocOpen = true
  images = ["/banner.png"]

[params.assets]
  favicon = "/favicon.png"
  css = [
    "css/extended/theme-vars.css",
    "css/extended/about.css"
  ]

[[menu.main]]
  name = "About"
  url = "/about/"
  weight = 1

deploy.yml – GitHub Pages Deployment#

name: Deploy Hugo site to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
      - run: hugo --minify
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
          external_repository: yourname/yourname.github.io
          publish_branch: main
          cname: example.com

🖼 Replacing Branding#

Swap out these in /static/:

FilePurpose
banner.pngHomepage hero
favicon.pngTab icon
about.pngAbout page portrait
logo.pngOptional UI mark

Customize your own layout styles via /assets/css/extended/.

✅ Build & Deploy Steps#

  1. Clone this structure into a new repo

  2. Update:

    • config.toml
    • GitHub workflow targets
    • Content files

  3. Push to main

  4. GitHub Actions will deploy to your Pages repo

🔁 Reuse Scenarios#

This setup works well for:

  • New personal sites or microsites
  • Project-specific documentation
  • Internal wikis or knowledge bases
  • Lightweight marketing pages with dark/light support

✅ Summary#

This Hugo blueprint combines:

  • PaperMod’s speed and structure
  • GitHub Actions for deployment
  • A modular CSS override system
  • Leaf bundles for scale-ready project docs

It’s clean, extensible, and ready for cloning.
